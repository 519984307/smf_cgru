Afanasy MS Windows Client

There is no server for windows. There is no reason to translate sources or launch with emulators current Linux version. Any (even very small) company can have at least one Linux server for various purposes (web, forum, wiki, ftp, mail, file server, backup, router, ... ). You can use it for Afanasy too.

Windows release contains client applications: render, watch, talk and python module to send jobs.

  1. Get CGRU

    Download the latest release and unpack.

    ( You don't need to install Qt. Qt's libraries exist with CGRU archive. They are patched! Do not use them for other purposes. )

  2. Install python2.5

    Afanasy is built with Python 2.5 version. To use other python versions you can download bin_pyaf archive. It contains pyaf.pyd Afanasy plugin for various Python versions, bits and OS'es.

  3. Launch

    Afanasy applications launch scripts are located in cgru/examples/scripts directory.
    afrender.cmd - render slave client.
    afwatch.cmd - GUI.
    aftalk.cmd - chat client.
    The binaries are located in afanasy/bin directory.

    Setup & Launch page shows how to run Afanasy applications and to use Python module to send jobs.

  4. Web Server

    If it installed on afanasy server, it available at

    http://[afanasy server computer name, or IP address]

    Or ask your admin configured it.

You can build the latest version yourself using MSVC. Dowload and install Qt. Use SVN (Subversion) to get code and CMake to build MSVC project files.
Go to afanasy/src/project.cmake folder.
Launch win_build_msvc.cmd to make MSVC project.
Launch win_build_msvc_pyaf.cmd to make MSVC project to build python module only.

Also you can build Afanasy using QMake or Ruby & Mxx_ru.
