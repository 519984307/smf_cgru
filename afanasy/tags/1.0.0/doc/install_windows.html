Afanasy MS Windows Client

Afanasy Windows version is 'alpha'. It appears only because Afanasy client applications are written on Qt, which is cross-platform (and build uses CMake, which is cross-platform too). There is no Afanasy developers using this platform (yet). So nobody testing it and fixing bugs.

There is no server for windows. Afanasy server uses some Linux common tools, for example Apache, SQL (Postgre), PHP. There is no reason to translate sources or launch with emulators current Linux version. Any (even very small) company can have at least one Linux server for various purposes (forum, web, ftp, mail, file server, router). You can use it for Afanasy too.

Latest release contains windows client applications: render, watch, talk and python module to send jobs. They built against a version of server which sources are in the same release.

You can build the latest version yourself using MSVC or MinGW. Dowload and install Qt. Use SVN (Subversion) to get code and CMake to build make files.

  1. Get CGRU

    Download latest release and unpack
    Better to do it once on server and share tools directory, so same directory will be on client

  2. Install python2.5

    Nuke, Houdini, Maya use Python 2.5 version.

  3. Launch

    Afanasy applications launch scripts are located in cgru/bin directory.
    afrender.cmd - render slave client.
    afwatch.cmd - GUI.
    aftalk.cmd - chat client.

  4. Web Server

    If it installed on afanasy server, it available at

    http://[afanasy server computer name]

    Or ask your admin configured it.
